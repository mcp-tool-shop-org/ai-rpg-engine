# Dogfood Swarm Audit — ai-rpg-engine v2.4.0 → v2.5.0

> **Role:** advisor audit (coordinator-authored). **Date:** 2026-07-07.
> **Baseline:** HEAD `a7450fc` (origin/main), v2.4.0, **3190 tests / 169 files green**, build clean.
> **Save-point:** tag `dogfood-save-1780446065`.
> **Class:** db-cluster-class swarm — take an already-shipped, already-green product *deeper*. This is NOT a broken-repo rescue.

## Method

Five read-only domain auditors (core+determinism · modules/combat · AI/ollama+image-gen · content/starters · CLI/CI/docs), each with a **severity-calibration discipline** (name the worst *realistic* consequence before assigning severity — the testing-os learning that auditors systematically over-rate). Every auditor ran a family-of-call-sites probe.

Findings were then **verified two ways** before landing here:
1. **Direct ground-truth** for command-checkable claims (greps, `npm pack --dry-run`, tag/release/npm state, test counts) — definitive.
2. **Cross-family cloud panel** (`ollama_verify_claims`, Ollama Cloud disjoint-family trio: deepseek-v4-pro / kimi-k2.7-code / glm-5.2) for the subtle code-reasoning claims — the EXTERNAL_VERIFIER standard (different family, generator reasoning stripped by schema). All three models substantively confirmed the five submitted claims; deepseek's verdicts parsed cleanly, the other two confirmed but were auto-excluded on a JSON-fence formatting artifact (not disagreement).

**Two findings were REJECTED by verification** (auditor over-reach caught):
- **G1 "no v2.4.0 git tag"** — FALSE. The tag, GitHub release, and npm `latest`=2.4.0 all exist; the auditor (and my own first pass) read a stale local clone that hadn't `git fetch --tags`. SHIP_GATE's "version matches tag" row is **true**.
- **"create-starter accepts a whitespace name"** — NOT a current bug. The CLI validates `^[a-z][a-z0-9]*(-[a-z0-9]+)*$` (well-tested). The `"   "` root dir was v2.3.6 forensic debris (untracked, `@2.3.6`), now deleted.

## Verdict

**The product's load-bearing claims hold.** Determinism (byte-identical replay), the "AI never mutates simulation truth" boundary, and "all 10 starters use `buildCombatStack()`" are all **true in code** (see Positives). The real defects are (a) a cluster of correctness bugs in the *newest* v2.4 party/status code, (b) a set of *quality-enforcement mechanisms that are themselves broken or vacuous* (a rubric check that inspects the wrong field, a template example that's a dead no-op, an integrity gate that only checks format), and (c) honesty-surface drift. None is a CRITICAL. Six are HIGH.

## The through-line: broken quality-enforcement mechanisms

Four of the sharpest findings share one shape — *the thing built to guarantee quality is itself broken, and its own test/gate is vacuous so nothing caught it*:

- **D2** — the template example authors copy is a dead no-op; its test asserts the initial value, never the effect.
- **D1** — the rubric dimension that's supposed to enforce faction topology inspects a dialogue module flag and prints a claim it never checked.
- **G2** — the docs-integrity gate that's supposed to catch version drift only checks semver format.
- **C1** — the extension API (`registerEffect`) that's supposed to let modules add rule effects stores them and never runs them.

This is the testing-os "vacuous gate" meta-pattern. The remediation discipline must be: **every fix ships a meta-test that mutates the protected thing and asserts the gate/mechanism actually fires** (delete a documented event → template test goes red; a factionless pack → rubric dimension fails; a registered effect → it actually executes).

## Positives (verified sound — do not re-litigate)

- **Determinism holds.** Core `src` is free of `Math.random`/`Date.now`/clock/entropy leaks; RNG is seeded mulberry32; the id counter is genuinely per-instance + serialized; event ordering is a stable total order. Byte-identical replay is true for the `submitAction` re-drive and serialize/deserialize paths (the only nuance is C4's mislabeled `getActionLog`).
- **"AI never mutates simulation truth" holds in code.** No `child_process`/`exec`/`eval`/`vm` sink anywhere in the three AI packages; output is stdout-by-default; the model writes only on explicit human confirm through the sandboxed `applyConfirmed`; `loadContent()` re-validates everything at load. Two honest doc-vs-code gaps (A2, A3), neither a silent mutation.
- **`chat-webfetch.ts` SSRF defense is exemplary** (blocks loopback/link-local/CGNAT/IPv6-tunnelled-v4/cloud-metadata; opt-in; external content fenced) — keep as the reference pattern.
- **All 10 starters genuinely use `buildCombatStack()`** and export the full CLI contract.
- **terminal-ui can't strand the user** (line-based readline, nothing calls `setRawMode`); saves are versioned + guarded (campaign-memory, character-profile v1→v2 migration); corrupt-save handling is structured.
- **Combat math is safe** — damage/hit formulas clamp negative/zero stats; selectors + `SeededRNG.int` guard empty/inverted/NaN; single-chain reflect terminates; DoT/HoT expiry is deterministic.
- **CI is sound** — paths-gated, concurrency-guarded, 20/22 matrix; `release.yml` is OIDC + idempotent + test-gated; the doc-examples typecheck genuinely guards README snippets against API drift.
