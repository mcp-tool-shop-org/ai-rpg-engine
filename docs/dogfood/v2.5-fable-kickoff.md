## Amend mechanics (Protocol Phase 3 + v2 amend, per dogfood-swarm.md)

- **Exclusive file ownership by domain (Principle 1).** No agent/edit crosses a domain boundary. If you spawn domain sub-agents, use `--isolate`-equivalent discipline; if you fix serially yourself, keep the domain map.
- **Test-first gate per finding (v2).** Write the failing test that probes the *full named invariant*, confirm it FAILS at HEAD, fix, confirm PASS, report the test id.
- **Meta-test where the finding is a vacuous mechanism (C1, D1, D2).** The test must *mutate the protected thing and assert the mechanism fires* — a registered effect actually executes; a factionless pack FAILS the rubric dimension; deleting the template listener turns its test RED. "Passes N/N" ≠ "works."
- **Family-of-call-sites probe.** After the named site, grep every sibling and fix the family (M2: every offensive resolver; A5: every public method building a path from `hash`).
- **Build-green barrier after the wave (Principle 4):** `npm run build && npm run test` green, run 3× if anything looks flaky.
- **Determinism guardrail (core/modules):** no `Date.now`/`Math.random` in resolution; ticks from the engine counter; RNG only via seeded `world.rng`; new ids via `genId(world, prefix)` (never global `nextId`); total stable tie-break keys.

**M1 (HIGH, Modules)** — `modules/src/status-effects.ts:369,388` reactive-trigger `chainDepth` accumulates across an AoE's damage events. **Invariant:** identical reactive statuses on N entities hit by one AoE produce identical reflect/thorns counts regardless of entity order; the depth cap still stops a single ping-pong chain. **Test-first:** two identical spiked-armor defenders + one enemy AoE; assert equal reflect counts AND order-invariance (fails today). **Fix:** seed each event's queue at `depth:0`, keep the *shared* already-fired dedup Set.
