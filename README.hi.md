# AI आरपीजी इंजन

आरपीजी दुनियाओं को बनाने, विश्लेषण करने और संतुलित करने के लिए एक सिमुलेशन-आधारित टूलकिट।

AI आरपीजी इंजन एक नियतात्मक सिमुलेशन रनटाइम को एक AI-सहायता प्राप्त डिज़ाइन स्टूडियो के साथ जोड़ता है, ताकि लेखक दुनिया बना सकें, सिमुलेशन के माध्यम से उनका परीक्षण कर सकें, और अनुमान लगाने के बजाय, साक्ष्यों के आधार पर उन्हें बेहतर बना सकें।

> पारंपरिक उपकरण आपको कहानियाँ लिखने में मदद करते हैं।
> AI आरपीजी इंजन आपको **दुनियाओं का परीक्षण** करने में मदद करता है।

---

## यह क्या करता है

```
build → critique → simulate → analyze → tune → experiment
```

आप दुनिया की सामग्री उत्पन्न कर सकते हैं, डिज़ाइनों की आलोचना कर सकते हैं, नियतात्मक सिमुलेशन चला सकते हैं, रीप्ले व्यवहार का विश्लेषण कर सकते हैं, यांत्रिकी को समायोजित कर सकते हैं, कई प्रारंभिक बिंदुओं पर प्रयोग चला सकते हैं, और परिणामों की तुलना कर सकते हैं। प्रत्येक परिणाम पुन: प्रस्तुत करने योग्य, जांचने योग्य और समझने योग्य है।

---

## मुख्य विशेषताएं

### नियतात्मक सिमुलेशन

आरपीजी दुनियाओं के लिए एक टिक-आधारित सिमुलेशन इंजन। दुनिया की स्थिति, घटना प्रणाली, धारणा और अनुभूति परतें, गुटों का विश्वास प्रसार, अफवाह प्रणाली, जिले के आंकड़े, पुन: चलाने योग्य क्रिया लॉग, और नियतात्मक रैंडम नंबर जनरेटर। प्रत्येक रन को बिल्कुल वैसा ही फिर से चलाया जा सकता है।

### AI-सहायता प्राप्त दुनिया निर्माण

एक वैकल्पिक AI परत जो एक थीम से कमरे, गुट, खोज और जिलों को बनाती है। यह डिज़ाइनों की आलोचना करता है, स्कीमा त्रुटियों को ठीक करता है, सुधारों का सुझाव देता है, और दुनिया निर्माण की बहु-चरणीय प्रक्रियाओं का मार्गदर्शन करता है। AI सीधे सिमुलेशन स्थिति को कभी नहीं बदलता है - यह केवल सामग्री या सुझाव उत्पन्न करता है।

### मार्गदर्शित डिज़ाइन प्रक्रियाएं

सत्र-जागरूक, योजना-आधारित प्रक्रियाएं दुनिया के निर्माण, आलोचना लूप, डिज़ाइन पुनरावृत्ति, निर्देशित निर्माण और संरचित ट्यूनिंग योजनाओं के लिए। यह नियतात्मक उपकरणों को AI सहायता के साथ जोड़ता है।

### सिमुलेशन विश्लेषण

रीप्ले विश्लेषण जो यह बताता है कि घटनाएं क्यों हुईं, यांत्रिकी कहाँ विफल होते हैं, कौन से ट्रिगर कभी नहीं चलते हैं, और कौन सी प्रणालियाँ अस्थिरता पैदा करती हैं। संरचित निष्कर्ष सीधे ट्यूनिंग में फीड किए जाते हैं।

### मार्गदर्शित ट्यूनिंग

संतुलन निष्कर्ष संरचित ट्यूनिंग योजनाओं को प्रस्तावित सुधारों, अपेक्षित प्रभाव, आत्मविश्वास अनुमानों और पूर्वावलोकन किए गए परिवर्तनों के साथ उत्पन्न करते हैं। यह पूरी ट्रेसबिलिटी के साथ, चरण-दर-चरण लागू किया जाता है।

### परिदृश्य प्रयोग

सामान्य व्यवहार को समझने के लिए प्रारंभिक बिंदुओं पर सिमुलेशन के बैच चलाएं। परिदृश्य मेट्रिक्स निकालें, विचलन का पता लगाएं, मापदंडों को बदलें, और ट्यून किए गए बनाम बेसलाइन दुनिया की तुलना करें। यह दुनिया के डिज़ाइन को एक परीक्षण योग्य प्रक्रिया में बदल देता है।

### स्टूडियो शेल

CLI डिज़ाइन स्टूडियो जिसमें प्रोजेक्ट डैशबोर्ड, समस्या ब्राउज़िंग, प्रयोग निरीक्षण, सत्र इतिहास, निर्देशित ऑनबोर्डिंग और संदर्भ-जागरूक कमांड खोज शामिल हैं। यह दुनिया बनाने और परीक्षण करने के लिए एक कार्यक्षेत्र है।

## शुरुआत कैसे करें

```bash
# Install the CLI
npm install -g @ai-rpg-engine/cli

# Start the interactive studio
ai chat

# Run onboarding
/onboard

# Create your first content
create-room haunted chapel

# Run a simulation
simulate

# Analyze the results
analyze-balance

# Tune the design
tune paranoia

# Run an experiment
experiment run --runs 50
```

## उदाहरण कार्यप्रवाह

## दर्शन

एआई आरपीजी इंजन तीन विचारों पर आधारित है:

1. **निर्धारित दुनिया** - सिमुलेशन के परिणाम पुन: उत्पन्न किए जाने योग्य होने चाहिए।
2. **साक्ष्य-आधारित डिज़ाइन** - दुनिया की कार्यप्रणाली का परीक्षण सिमुलेशन के माध्यम से किया जाना चाहिए।
3. **एआई एक सहायक है, अधिकार नहीं** - एआई उपकरण डिज़ाइनों को उत्पन्न करने और उनका मूल्यांकन करने में मदद करते हैं, लेकिन वे निर्धारित प्रणालियों को प्रतिस्थापित नहीं करते हैं।

पूर्ण विवरण के लिए [PHILOSOPHY.md](PHILOSOPHY.md) देखें।

## सुरक्षा

एआई आरपीजी इंजन एक **स्थानीय सिमुलेशन लाइब्रेरी** है। इसमें कोई टेलीमेट्री, कोई नेटवर्क और कोई गुप्त जानकारी नहीं है। सेव फाइलें केवल `.ai-rpg-engine/` फ़ोल्डर में जाती हैं, जब स्पष्ट रूप से अनुरोध किया जाता है। विवरण के लिए [SECURITY.md](SECURITY.md) देखें।
