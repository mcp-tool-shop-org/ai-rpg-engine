# एआई आरपीजी इंजन

नियतात्मक आरपीजी सिमुलेशन बनाने के लिए एक टाइपस्क्रिप्ट टूलकिट। आप आँकड़े परिभाषित करते हैं, मॉड्यूल चुनते हैं, युद्ध प्रणाली को जोड़ते हैं और सामग्री बनाते हैं। इंजन स्थिति, घटनाओं, यादृच्छिक संख्या जनरेटर (आरएनजी), क्रिया समाधान और एआई निर्णय लेने का प्रबंधन करता है। प्रत्येक रन दोहराया जा सकता है।

यह एक **कंपोज़िशन इंजन** है, न कि कोई तैयार गेम। 10 शुरुआती दुनिया उदाहरण हैं - विघटनीय पैटर्न जिनसे आप सीखते हैं और उन्हें फिर से जोड़ते हैं। आपके गेम में इंजन के जो भी उपसमुच्चय की आवश्यकता होती है, उसका उपयोग किया जाता है।

---

## यह क्या है

- एक **मॉड्यूल लाइब्रेरी** - 30+ इंजन मॉड्यूल जिसमें युद्ध, धारणा, अनुभूति, गुट, अफवाहें, यात्रा, साथी और बहुत कुछ शामिल हैं।
- एक **कंपोज़िशन टूलकिट** - `buildCombatStack()` लगभग 7 लाइनों में युद्ध को जोड़ता है; `new Engine({ modules })` गेम शुरू करता है।
- एक **सिमुलेशन रनटाइम** - नियतात्मक टिक, दोहराए जा सकने वाले क्रिया लॉग, सीडेड आरएनजी।
- एक **एआई डिज़ाइन स्टूडियो** (वैकल्पिक) - ढांचा, आलोचना, संतुलन विश्लेषण, ट्यूनिंग, ओलामा के माध्यम से प्रयोग।

## यह क्या नहीं है

- यह तुरंत खेलने योग्य गेम नहीं है - आप मॉड्यूल और सामग्री से इसे बनाते हैं।
- यह एक दृश्य इंजन नहीं है - यह संरचित घटनाओं को आउटपुट करता है, पिक्सेल नहीं।
- यह एक कहानी जनरेटर नहीं है - यह दुनिया का अनुकरण करता है; कथा यांत्रिकी से उभरती है।

## वर्तमान स्थिति (v2.5.0)

**जो काम करता है और परीक्षण किया गया है:**
- मुख्य रनटाइम: विश्व स्थिति, घटनाएं, क्रियाएं, टिक, पुनरावृत्ति - v1.0 से स्थिर; नियतात्मक बाइट-समान पुनरावृत्ति (प्रति-उदाहरण आईडी काउंटर, सीडेड आरएनजी)।
- युद्ध प्रणाली: 5 क्रियाएं, 4 युद्ध अवस्थाएं, 4 संलग्नता अवस्थाएं, साथी अवरोधन, हार का प्रवाह, एआई रणनीति।
- क्षमताएं: लागत, कूलडाउन, आँकड़े की जांच, टाइप किए गए प्रभाव, 11-टैग स्थिति शब्दावली, एआई-जागरूक चयन।
- **पार्टी युद्ध (v2.4):** सहयोगी-लक्ष्यीकरण (उपचार / बफ़ / पुनर्जीवित), मित्र / शत्रु AoE फ़िल्टरिंग, लक्ष्य चयनकर्ता - एक चिकित्सक एक टीम के साथी का इलाज कर सकता है; दुश्मन AoE सहयोगियों को छोड़ देता है।
- **स्थिति प्रभाव (v2.4):** निष्क्रिय आँकड़े संशोधक युद्ध तक पहुंचते हैं, नियतात्मक DoT/HoT टिक काउंटर से, गहराई-सीमित प्रतिक्रियाशील ट्रिगर (कांटे / परावर्त)।
- **प्लग-इन प्रोफाइल - प्रति-इकाई नियम समाधान (v2.5):** एक `might` योद्धा और एक `will` रहस्यवादी एक लड़ाई में युद्ध को हल करते हैं, प्रत्येक अपनी मैपिंग के माध्यम से आँकड़े पढ़ते हैं। `RuleProfile` + `WorldState.ruleProfiles` + `EntityState.ruleProfileId`; `applyProfile()` एक प्रोफ़ाइल संलग्न करता है (आँकड़ा मैपिंग, संसाधन पूल, प्रति-इकाई क्षमताएं); `buildProfile()`, `validateProfileSet()` (डुप्लिकेट आईडी अस्वीकृत), 10 शुरुआती व्युत्पन्न टेम्पलेट और एक `profile` CLI कमांड।
- एकीकृत निर्णय परत: युद्ध + क्षमता स्कोरिंग को एक कॉल में मिलाया गया (`selectBestAction`)।
- सभी 10 शुरुआती दुनिया `buildCombatStack()` का उपयोग करती हैं - सिद्ध रचना रीढ़ की हड्डी।
- अनुभूति कॉन्फ़िगरेशन एपीआई (`cognition: CognitionCoreConfig | false`) प्रति-शुरुआती एआई ट्यूनिंग के लिए।
- सामग्री निर्माण के लिए टैग टैक्सोनॉमी और सत्यापन उपकरण।
- `ai-rpg-engine create-starter <name>` - एक नया गेम बनाएं; `validate` + `scaffold` सामग्री कमांड; JSON से पैक लोड करें।
- npm पर प्रकाशित शुरुआती टेम्पलेट (`@ai-rpg-engine/starter-template`)।
- पूर्ण परीक्षण सूट: **193 फ़ाइलों में 3613 परीक्षण** (दोहराए जाने वाले रनों में नियतात्मक; CI में कवरेज रैचेट-प्रवर्तित)।

**जो अधूरा या अपूर्ण है:**
- एआई विश्व निर्माण उपकरण (ओलामा परत) सिमुलेशन कोर की तुलना में कम परीक्षण किए गए हैं - हालांकि v2.5 ने संरचित त्रुटि प्रबंधन, एक कॉन्फ़िगर करने योग्य / देखने योग्य पुन: प्रयास लूप और उत्पन्न सामग्री पर एक वैकल्पिक `--validate` गेट जोड़ा।
- मल्टीप्लेयर (एक ही दुनिया साझा करने वाले दो मानव खिलाड़ी) बनाया नहीं गया है - यह एक नेटवर्किंग परत है, जानबूझकर दायरे से बाहर; प्रोफाइल आज एक एकल नियंत्रक को लक्षित करते हैं।
- प्रलेखन व्यापक है लेकिन हर हैंडबुक पृष्ठ नवीनतम एपीआई को प्रतिबिंबित नहीं करता है।

## त्वरित शुरुआत

```typescript
import { Engine } from '@ai-rpg-engine/core';
import { buildCombatStack, traversalCore, statusCore, createDialogueCore } from '@ai-rpg-engine/modules';

// Define your stat mapping
const combat = buildCombatStack({
  statMapping: { attack: 'might', precision: 'agility', resolve: 'will' },
  playerId: 'hero',
  biasTags: ['undead', 'beast'],
});

// Wire the engine
const engine = new Engine({
  manifest: myManifest,
  modules: [statusCore, ...combat.modules, traversalCore, createDialogueCore(myDialogues)],
});

// Submit player actions
engine.submitAction('attack', { targetIds: ['skeleton-1'] });

// Submit AI entity actions
engine.submitActionAs('guard-captain', 'attack', { targetIds: ['player'] });
```

पूर्ण वर्कफ़्लो के लिए [कंपोज़िशन गाइड](docs/handbook/57-composition-guide.md) देखें, या एक नया शुरुआती बनाएं:

```bash
npx @ai-rpg-engine/cli create-starter my-game
```

## युद्ध प्रणाली

पांच क्रियाएं (हमला, रक्षा, अलग होना, सहारा देना, पुन: स्थिति), चार युद्ध अवस्थाएं (संरक्षित, असंतुलित, उजागर, भागना), चार संलग्नता अवस्थाएं (संलग्न, संरक्षित, बैकलाइन, पृथक)। तीन आँकड़े आयाम हर सूत्र को चलाते हैं इसलिए एक त्वरित द्वंद्ववादी एक भारी ब्रूज़र या एक रचनाबद्ध प्रहरी से अलग तरीके से खेलता है।

एआई विरोधी एकीकृत निर्णय स्कोरिंग का उपयोग करते हैं - युद्ध क्रियाएं और क्षमताएं एक ही मूल्यांकन में प्रतिस्पर्धा करती हैं, जिसमें मामूली क्षमता स्पैम को रोकने के लिए कॉन्फ़िगर करने योग्य सीमाएँ होती हैं।

पैकेज लेखक `buildCombatStack()` का उपयोग आँकड़ा मैपिंग, संसाधन प्रोफ़ाइल और पूर्वाग्रह टैग से युद्ध को जोड़ते हैं। [युद्ध अवलोकन](docs/handbook/49a-combat-overview.md) और [पैकेज लेखक गाइड](docs/handbook/55-combat-pack-guide.md) देखें।

## क्षमताएं

शैली-देशी क्षमता प्रणाली जिसमें लागत, आँकड़े की जांच, कूलडाउन और टाइप किए गए प्रभाव (नुकसान, उपचार, स्थिति लागू करें, शुद्ध करें) शामिल हैं। स्थिति प्रभावों में प्रतिरोध / भेद्यता प्रोफाइल के साथ 11-टैग सिमेंटिक शब्दावली का उपयोग किया जाता है। एआई-जागरूक चयन स्व / AoE / एकल-लक्ष्य पथों को स्कोर करता है।

```typescript
const warCry: AbilityDefinition = {
  id: 'war-cry', name: 'War Cry', verb: 'use-ability',
  tags: ['combat', 'debuff', 'aoe'],
  costs: [{ resourceId: 'stamina', amount: 3 }],
  target: { type: 'all-enemies' },
  checks: [{ stat: 'nerve', difficulty: 6, onFail: 'abort' }],
  effects: [
    { type: 'apply-status', target: 'target', params: { statusId: 'rattled', duration: 2 } },
  ],
  cooldown: 4,
};
```

### शुरुआती उदाहरण

ये दस शुरुआती दुनियाएँ **रचना के उदाहरण** हैं – ये दर्शाती हैं कि गेम इंजन मॉड्यूल को मिलाकर पूर्ण गेम कैसे बनाया जा सकता है। प्रत्येक दुनिया विभिन्न प्रकार के पैटर्न (सांख्यिकीय मानचित्रण, संसाधन प्रोफाइल, जुड़ाव कॉन्फ़िगरेशन, क्षमता सेट) दिखाती है। प्रत्येक शुरुआती दुनिया के ‘रीडमी’ में “दिखाए गए पैटर्न” और “क्या उपयोग किया जा सकता है” देखें।

| शुरुआती/प्रारंभिक | शैली | प्रमुख पैटर्न |
|---------|-------|-------------|

## कार्य योजना

### हम अभी कहाँ हैं।

सिमुलेशन रनटाइम, युद्ध संरचना का आधार, और शुरुआती लेखक मार्ग पूर्ण हो गया है – कुल 193 फ़ाइलों में 3613 परीक्षण किए गए, ‘बिल्डकॉम्बैटस्टैक’ पर सभी 10 शुरुआती सेटिंग्स लागू की गईं, निश्चित रूप से समान बाइट-दर-बाइट रीप्ले सुनिश्चित किया गया, पूरी तरह से एआई निर्णय स्कोरिंग की गई, और एक सीएलआई ढांचा कमांड जोड़ा गया। **संस्करण 2.5 प्रति इकाई नियम समाधान प्रदान करता है – प्रमुख प्लग-इन प्रोफाइल सुविधा: एक ‘शक्तिशाली’ योद्धा और एक ‘दृढ़ निश्चयी’ रहस्यवादी एक ही लड़ाई में युद्ध का संचालन करते हैं, प्रत्येक अपनी स्वयं की मैपिंग के माध्यम से आँकड़ों को पढ़ता है।**

**नवीनतम संस्करण श्रृंखला (v2.3.3–v2.5.0):**
- v2.3.3–v2.3.7 — उपभोक्ता कलाकृति का सत्यापन, युद्ध प्रणाली को मजबूत करना, `buildCombatStack` में सभी 10 शुरुआती तत्वों को शामिल करना, प्रारंभिक टेम्पलेट प्रकाशित करना, `create-starter` कमांड लाइन इंटरफ़ेस (सीएलआई)
- v2.4.0 — पार्टी युद्ध (सहयोगी-लक्ष्यीकरण / उपचार / शक्तिवर्धन / पुनर्जीवन, मित्र-शत्रु क्षेत्र प्रभाव), स्थिति-प्रभाव प्रणाली (संशोधक + क्षति/उपचार समय के साथ + प्रतिक्रियाशील ट्रिगर), प्लग-इन प्रोफाइल चरण 1, सामग्री `validate`/`scaffold` सीएलआई
- **v2.5.0 — प्रति-इकाई नियम समाधान (मिश्रित-खेल शैली युद्ध), `applyProfile` लोडर + प्रति-इकाई क्षमताएं, प्रोफ़ाइल टेम्पलेट + `profile` सीएलआई, और पूर्ण स्वास्थ्य जांच (बाइट-समान-पुनरावृत्ति सुधार, सटीकता को मजबूत करना, गुणवत्ता नियंत्रण लागू)**

### अगला।

- मल्टीप्लेयर – दो *मानव* खिलाड़ी एक ही दुनिया साझा करते हैं (एक नेटवर्किंग परत, जिसे जानबूझकर स्थगित किया गया है; एकल नियंत्रक द्वारा साझा किए गए प्रोफाइल आज जारी किए जा रहे हैं, जैसे [`shared-profiles.ts`](docs/examples/shared-profiles.ts))
- क्रमबद्ध सूत्र अधिलेखन – प्रति-प्रोफ़ाइल सूत्र ट्यूनिंग (एक सूत्र डीएसएल पर निर्भर; प्रोफ़ाइल में आज आँकड़ों का मानचित्रण होता है, क्लोजर नहीं)
- एपीआई प्रलेखन सिंक्रनाइज़ेशन – सुनिश्चित करें कि प्रत्येक निर्देशिका पृष्ठ v2.5 एपीआई को दर्शाता है।

### गंतव्य: प्लग-इन प्रोफाइल।

इंजन का अंतिम लक्ष्य **उपयोगकर्ता द्वारा परिभाषित प्रोफाइल** बनाना है – ये पोर्टेबल पैकेज होते हैं जिन्हें किसी भी गेम में इस्तेमाल किया जा सकता है। एक प्रोफाइल आँकड़ों का मानचित्रण, संसाधनों का व्यवहार, कृत्रिम बुद्धिमत्ता (एआई) पूर्वाग्रह टैग और क्षमताओं को एक ही आयात योग्य इकाई में समेटता है। संस्करण 2.5 के अनुसार, एक दुनिया में मौजूद प्रत्येक वस्तु अपना स्वयं का प्रोफाइल रख सकती है और प्रति-वस्तु युद्ध को हल कर सकती है – उदाहरण के लिए, एक ‘शक्तिशाली’ योद्धा और एक ‘दृढ़ निश्चयी’ रहस्यवादी एक साथ मिलकर काम करते हैं, जिनमें से प्रत्येक अपनी अनूठी खेल शैली लाता है।

स्कीमा, `applyProfile` लोडर, प्रति-एंटिटी क्षमता समाधान और क्रॉस-प्रोफ़ाइल सत्यापन – ये सभी अब उपलब्ध हैं। शेष केवल मल्टीप्लेयर सुविधा है — जिसमें दो *मानव* खिलाड़ी (सिर्फ़ दो एंटिटी नहीं) एक साथ एक दुनिया साझा कर सकते हैं — जो कि एक नेटवर्किंग परत है। डिज़ाइन के लिए [प्रोफ़ाइल रोडमैप](docs/profile-roadmap.md) और [फ़ीचर-आर्किटेक्चर.md](docs/feature-architecture.md) देखें।

## दर्शनशास्त्र

एआई आरपीजी इंजन तीन मुख्य विचारों पर आधारित है:

1. **निश्चित दुनिया** — सिमुलेशन के परिणाम दोहराए जा सकने चाहिए।
2. **साक्ष्य-आधारित डिज़ाइन** — दुनिया की यांत्रिकी का परीक्षण सिमुलेशन के माध्यम से किया जाना चाहिए।
3. **सहायक के रूप में एआई, अधिकार नहीं** — एआई उपकरण डिज़ाइनों को उत्पन्न करने और उनकी आलोचना करने में मदद करते हैं, लेकिन वे निश्चित प्रणालियों को प्रतिस्थापित नहीं करते हैं।

पूर्ण विवरण के लिए [PHILOSOPHY.md](PHILOSOPHY.md) देखें।
