# कृत्रिम बुद्धिमत्ता (एआई) आधारित रोल-प्लेइंग गेम इंजन।

टाइपस्क्रिप्ट का एक टूलकिट, जिसका उपयोग नियतात्मक आरपीजी सिमुलेशन बनाने के लिए किया जाता है। आप आँकड़े परिभाषित करते हैं, मॉड्यूल चुनते हैं, एक युद्ध प्रणाली बनाते हैं, और सामग्री बनाते हैं। इंजन स्थिति, घटनाओं, यादृच्छिक संख्या जनरेटर (आरएनजी), क्रियाओं का समाधान और कृत्रिम बुद्धिमत्ता (एआई) निर्णय लेने का प्रबंधन करता है। प्रत्येक रन को दोहराया जा सकता है।

यह एक **रचना इंजन** है, कोई तैयार गेम नहीं। 10 शुरुआती दुनिया उदाहरण हैं - विघटित पैटर्न जिनसे आप सीखते हैं और जिन्हें आप फिर से उपयोग करते हैं। आपका गेम इंजन के उन हिस्सों का उपयोग करता है जिनकी आपको आवश्यकता होती है।

---

## यह क्या है

- एक **मॉड्यूल लाइब्रेरी** - 27 से अधिक मॉड्यूल जो युद्ध, धारणा, अनुभूति, गुटों, अफवाहों, यात्रा, साथियों और अन्य चीज़ों को कवर करते हैं।
- एक **रचना टूलकिट** - `buildCombatStack()` लगभग 7 पंक्तियों में युद्ध प्रणाली को जोड़ता है; `new Engine({ modules })` गेम शुरू करता है।
- एक **सिमुलेशन रनटाइम** - नियतात्मक गणना, दोहराने योग्य क्रिया लॉग, सीडेड आरएनजी।
- एक **एआई डिज़ाइन स्टूडियो** (वैकल्पिक) - ढांचा, आलोचना, संतुलन विश्लेषण, ट्यूनिंग, ओलामा के माध्यम से प्रयोग।

## यह क्या नहीं है

- यह कोई ऐसा गेम नहीं है जिसे सीधे खेला जा सके - आप मॉड्यूल और सामग्री से इसे बनाते हैं।
- यह कोई दृश्य इंजन नहीं है - यह संरचित घटनाओं का आउटपुट देता है, पिक्सेल का नहीं।
- यह कोई कहानी जनरेटर नहीं है - यह दुनिया का अनुकरण करता है; कथा यांत्रिकी से उत्पन्न होती है।

## वर्तमान स्थिति (v2.3.0)

**जो काम करता है और जिसका परीक्षण किया गया है:**
- कोर रनटाइम: दुनिया की स्थिति, घटनाएं, क्रियाएं, गणना, दोहराव - v1.0 से स्थिर।
- युद्ध प्रणाली: 5 क्रियाएं, 4 युद्ध अवस्थाएं, 4 संलग्नक अवस्थाएं, साथी हस्तक्षेप, हार का क्रम, एआई रणनीति - 1099 परीक्षण।
- क्षमताएं: लागत, कूलडाउन, आँकड़े जांच, टाइप किए गए प्रभाव, स्थिति शब्दावली, एआई-जागरूक चयन।
- एकीकृत निर्णय परत: युद्ध + क्षमता स्कोरिंग को एक ही कॉल (`selectBestAction`) में मिलाया गया।
- 10 शुरुआती दुनिया, जिनमें आँकड़ों में भिन्न दुश्मन और पूर्ण युद्ध एकीकरण है।
- `buildCombatStack()` प्रति दुनिया लगभग 40 पंक्तियों की युद्ध सेटअप को समाप्त करता है।
- सामग्री निर्माण के लिए टैग वर्गीकरण और सत्यापन उपयोगिताएं।
- क्रॉस-फेज टैग ट्रेसिंग के साथ बॉस फेज सत्यापन।

**जो अधूरा या अपूर्ण है:**
- एआई दुनिया निर्माण उपकरण (ओलामा परत) काम करते हैं लेकिन सिमुलेशन की तुलना में इनका परीक्षण कम किया गया है।
- सीएलआई स्टूडियो शेल कार्यात्मक है लेकिन परिष्कृत नहीं है।
- केवल 10 शुरुआती दुनिया में से 1 (`वीर्ड वेस्ट`) `buildCombatStack` का उपयोग करता है; अन्य विस्तृत मैनुअल वायरिंग का उपयोग करते हैं।
- अभी तक कोई प्रोफाइल सिस्टम नहीं है - दुनिया स्टैंडअलोन हैं, साझा प्रोफाइल से बनाए नहीं जा सकते।
- प्रलेखन व्यापक है (57 अध्याय), लेकिन सभी अध्याय नवीनतम एपीआई को प्रतिबिंबित नहीं करते हैं।

## शुरुआत कैसे करें

```typescript
import { Engine } from '@ai-rpg-engine/core';
import { buildCombatStack, createTraversalCore, createDialogueCore } from '@ai-rpg-engine/modules';

// Define your stat mapping
const combat = buildCombatStack({
  statMapping: { attack: 'might', precision: 'agility', resolve: 'will' },
  playerId: 'hero',
  biasTags: ['undead', 'beast'],
});

// Wire the engine
const engine = new Engine({
  manifest: myManifest,
  modules: [...combat.modules, createTraversalCore(), createDialogueCore(myDialogues)],
});

// Submit player actions
engine.submitAction('attack', { targetIds: ['skeleton-1'] });

// Submit AI entity actions
engine.submitActionAs('guard-captain', 'attack', { targetIds: ['player'] });
```

पूर्ण कार्यप्रवाह के लिए [रचना गाइड](docs/handbook/57-composition-guide.md) देखें।

## युद्ध प्रणाली

पांच क्रियाएं (हमला, रक्षा, पीछे हटना, तैयार रहना, स्थिति बदलना), चार युद्ध अवस्थाएं (सुरक्षित, असंतुलित, उजागर, भागना), चार संलग्नक अवस्थाएं (जुड़ा हुआ, संरक्षित, पीछे की पंक्ति, अलग)। तीन आँकड़े आयाम प्रत्येक सूत्र को चलाते हैं, इसलिए एक त्वरित द्वंद्ववादी एक भारी योद्धा या एक शांत रक्षक से अलग तरीके से खेलता है।

एआई विरोधियों का उपयोग एकीकृत निर्णय स्कोरिंग से किया जाता है - युद्ध क्रियाएं और क्षमताएं एक ही मूल्यांकन में प्रतिस्पर्धा करती हैं, जिसमें मामूली क्षमताओं के अत्यधिक उपयोग को रोकने के लिए कॉन्फ़िगर करने योग्य सीमाएं होती हैं।

पैक लेखक युद्ध प्रणाली को एक आँकड़े मैपिंग, संसाधन प्रोफाइल और पूर्वाग्रह टैग से जोड़ने के लिए `buildCombatStack()` का उपयोग करते हैं। [युद्ध अवलोकन](docs/handbook/49a-combat-overview.md) और [पैक लेखक गाइड](docs/handbook/55-combat-pack-guide.md) देखें।

## क्षमताएं

शैली-विशिष्ट क्षमताओं का सिस्टम, जिसमें लागत, आँकड़े जांच, कूलडाउन और विभिन्न प्रकार के प्रभाव (नुकसान, उपचार, स्थिति परिवर्तन, शुद्धिकरण) शामिल हैं। स्थिति प्रभावों के लिए 11-टैग वाली शब्दसंग्रह प्रणाली का उपयोग किया जाता है, जिसमें प्रतिरोध/संवेदनशीलता प्रोफाइल शामिल हैं। कृत्रिम बुद्धिमत्ता (एआई) द्वारा संचालित चयन प्रणाली, स्व-नुकसान/क्षेत्रीय प्रभाव/एकल-लक्ष्य पथों का चयन करती है।

```typescript
const warCry: AbilityDefinition = {
  id: 'war-cry', name: 'War Cry', verb: 'use-ability',
  tags: ['combat', 'debuff', 'aoe'],
  costs: [{ resourceId: 'stamina', amount: 3 }],
  target: { type: 'all-enemies' },
  checks: [{ stat: 'nerve', difficulty: 6, onFail: 'abort' }],
  effects: [
    { type: 'apply-status', target: 'target', params: { statusId: 'rattled', duration: 2 } },
  ],
  cooldown: 4,
};
```

## रोडमैप।

### हम अभी कहां हैं।

सिमुलेशन रनटाइम और युद्ध प्रणाली मजबूत हैं - 2661 परीक्षण, 10 शैली उदाहरण, नियतात्मक पुनरावृत्ति, पूर्ण एआई निर्णय स्कोरिंग। इंजन एक मॉड्यूलर टूलकिट के रूप में काम करता है: मॉड्यूल चुनें, आँकड़े परिभाषित करें, कनेक्ट करें, सामग्री बनाएं। दस्तावेज़ में प्रत्येक प्रणाली को शामिल किया गया है, लेकिन नवीनतम परिवर्धनों के लिए एपीआई सिंक की आवश्यकता है।

### आने वाले कुछ सप्ताह।

- शेष 9 शुरुआती उदाहरणों को `buildCombatStack` में स्थानांतरित करें (विचित्र पश्चिम संदर्भ है)।
- एपीआई दस्तावेज़ सिंक - `submitActionAs`, `selectBestAction`, `resourceCaps`, टैग वर्गीकरण।
- शुरुआती उदाहरणों के लिए "README" फ़ाइल को बेहतर बनाएं - "क्या उधार लेना है" और "पुन: उपयोग कैसे करें" के बारे में स्पष्ट मार्गदर्शन।
- क्रॉस-लिंकिंग - "README", मॉड्यूलर गाइड, उदाहरण और हैंडबुक को एक साथ जोड़ा जाएगा।

### लक्ष्य: प्लग-इन प्रोफाइल।

इंजन का अंतिम लक्ष्य **उपयोगकर्ता-परिभाषित प्रोफाइल** हैं - पोर्टेबल बंडल जो किसी भी गेम में उपयोग किए जा सकते हैं। एक प्रोफाइल में आँकड़े मैपिंग, संसाधन व्यवहार, एआई पूर्वाग्रह टैग, क्षमताएं और मुठभेड़ हुक शामिल होते हैं, जिन्हें एक ही आयात योग्य इकाई में पैक किया जाता है। दो खिलाड़ी अलग-अलग प्रोफाइल के साथ एक ही दुनिया साझा कर सकते हैं, प्रत्येक अपनी खेल शैली लाता है।

प्रोफाइल मॉड्यूलर संरचना (जो पहले से ही काम कर रही है) और एकीकृत निर्णय परत (v2.3.0 में जारी) पर आधारित हैं। शेष कार्य प्रोफाइल स्कीमा को परिभाषित करना, लोडर बनाना और क्रॉस-प्रोफाइल इंटरैक्शन को मान्य करना है। पूर्ण योजना के लिए [प्रोफाइल रोडमैप](docs/profile-roadmap.md) देखें।

## दर्शन।

AI RPG इंजन तीन विचारों पर आधारित है:

1. **निश्चित दुनियाएं** — सिमुलेशन के परिणाम पुनरुत्पादित होने चाहिए।
2. **साक्ष्य-आधारित डिज़ाइन** — दुनिया के यांत्रिकी का परीक्षण सिमुलेशन के माध्यम से किया जाना चाहिए।
3. **एआई सहायक है, अधिकार नहीं** — एआई उपकरण डिज़ाइनों को उत्पन्न करने और उनकी आलोचना करने में मदद करते हैं, लेकिन वे निश्चित प्रणालियों को प्रतिस्थापित नहीं करते हैं।

पूर्ण विवरण के लिए [PHILOSOPHY.md](PHILOSOPHY.md) देखें।

## सुरक्षा।

AI RPG इंजन एक **स्थानीय-केवल सिमुलेशन लाइब्रेरी** है। कोई टेलीमेट्री नहीं, कोई नेटवर्क नहीं, कोई रहस्य नहीं। सेव फाइलें केवल तभी `.ai-rpg-engine/` में जाती हैं जब स्पष्ट रूप से अनुरोध किया जाता है। विवरण के लिए [SECURITY.md](SECURITY.md) देखें।
